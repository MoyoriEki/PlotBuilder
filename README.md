# PlotBuilder

漫画・物語のプロットをグリッドスナップ式ブロックとして配置・組み換えするWebアプリ「プロットボード」。

- 本体: `plotboard.html`(単一HTMLファイル・ノービルド)
- 公開URL: https://moyorieki.github.io/PlotBuilder/plotboard.html (GitHub Pages / mainへのpushで自動デプロイ)

## 設計方針

- 軽量・直感優先。D&DはPointer Eventsによる自前実装
- **縦長ファースト**: モバイル(スマホ/タブレット縦持ち)を主要ユースケースとし、初期グリッドは縦長(6列×12行)。今後のUIも縦長画面を優先して設計する
- ローカル(IndexedDB)が常に一次ストレージ。Dropbox連携は任意(未接続でも全機能が使える)
- **表示倍率(ズーム)**: 標準は画面幅フィット(=100%)。PCなど広い画面ではマスが小さく端に寄りがちなので、右下のズームピル(− / 倍率 / ＋)で拡大縮小できる。倍率は`cell`(1マスのpx)に掛けるだけなのでD&D等の座標計算はそのまま整合する。ブロック内の文字・マーク・その場エディタも`grid`に載せた`--zoom`CSS変数(`font-size: calc(… * var(--zoom) * var(--fscale))`)で倍率に追従して拡大する。倍率は端末ローカル設定(localStorage`pb_settings`の`displayZoom`)で作品ファイルには紐づかず、端末ごとに最後の設定を保持する。PCは`Ctrl/⌘ + ＋/−/0`とCtrl+ホイールにも対応
- **ブロックの文字サイズ設定**: 設定(⚙)のスライダーで50〜200%(5%刻み)を数値指定。ドラッグ中も即時反映。`--fscale`CSS変数として`--zoom`に掛け合わせる。端末ローカル設定(`pb_settings`の`fontScale`、数値)で作品JSONには含めない。旧4段階のキー文字列(`small`等)は`FONT_SCALES`で数値へ読み替える
- **マーク(★/⚑/×)バッジ**: ブロック左上角にem基準でまたがらせ、`layoutBlock`がバッジ幅のフロートスペーサー(`.mark-pad`)でタイトル1行目だけを避けさせるので文字と重ならない(`text-indent`だと入りきらない文字が横にクリップされるためフロート方式)
- **プリセット色のカスタマイズ**: 設定(⚙)からプリセット8色それぞれの色を変更できる。ブロックは`class="color-<id>"`で色を受け取るので、上書き分だけを`<style>`に注入すればブロック・スウォッチが即座に再着色される(再描画不要)。背景色を1つ選ぶと枠線・文字色はHSLを暗くして自動導出。上書きは端末ローカル設定(`pb_settings`の`palette`)として全作品共通で保持し、色ごと/一括でデフォルトに戻せる(作品JSONには含めない)
- **タイトル/メモ統一(Phase 3.5〜)**: ブロックのテキストは単一の`text`フィールド(データモデルversion 2)。1行目をタイトルとして強調表示し、2行目以降はブロックの高さに入る分だけ表示する(ブロックを大きくする＝メモが多く見える)。編集はモーダルではなくブロックに重なるその場ポップオーバーエディタで行う

## データモデル version 3(Phase 4)

- ブロックの`thumb`(未使用)を廃止し、`images`(画像ファイル名の配列・空配列可)を追加。**配列の1枚目=ブロックに表示されるサムネイル**(エディタの「サムネに設定」で先頭に移動)
- マイグレーション: v2→v3は`images = thumb ? [thumb] : []`(thumbは常にnull運用だったため実質全ブロック`[]`)。v1→v2(`text = title + "\n" + memo`)からの段階変換。旧バージョンのデータ(ローカル・Dropbox)は読み込み時に自動変換し、変換前の生データはIndexedDBの`docs`ストアに`v1-backup:{タイトル名}`/`v2-backup:{タイトル名}`として退避(消失事故防止。Phase 2→3のマイグレーションと同方針)
- 設定(エディタの確定キー等)は端末ローカル(localStorage`pb_settings`)。作品JSONには含めない

## 画像の仕様(Phase 4・将来の自分向けメモ)

- **追加**: その場エディタ内の画像ストリップから。ファイル選択(＋ボタン)・クリップボード貼り付け(Ctrl+V / 長押しペースト)・ファイルD&D(ブロックまたはエディタへ)の3経路
- **パイプライン**: 追加時にCanvasで長辺1024px以下へ縮小 → WebPエンコード品質0.85(WebPエンコード非対応ブラウザはJPEGへフォールバックし拡張子も追従)。元画像はどこにも保持しない。ファイル名は`img-{ランダムID}.webp|.jpg`で、作品JSONにはファイル名参照のみ入る
- **保存先**: Dropbox `/PlotBoard/{タイトル名}/img/{ファイル名}`。バイナリの送受信はストレージブリッジ標準の`save`(Blob可)/`loadBinary`をそのまま使う(ブリッジ改造なし。ブリッジ未提供でplotboard.html側が直叩きするのはrevの`files/get_metadata`とGCの`files/list_folder`のみ)
- **キャッシュ**: IndexedDBの`images`ストアにBlobで保存(キー:`{タイトル名}/{ファイル名}`)。表示はキャッシュ優先→なければDropboxからダウンロードしてキャッシュ。オフラインで未キャッシュは「オフライン」プレースホルダ(エラーにしない)。表示はobjectURLを使い回してドラッグ中の再デコードを防ぐ
- **アップロード**: 追加時はまずIndexedDBに保存して即表示し、レコードの`pendingImgs`に積んでJSON同期と同じ遅延・リトライ系統でアップロード(オフライン追加→復帰後アップロードが成立)。JSONより先に画像を上げ、参照先が欠けないようにする
- **不要画像GC**: JSONのDropboxアップロード成功直後のみ実行。いま上げたJSONのどのブロックの`images`にも含まれない`/img/`内ファイルをDropboxとIndexedDBキャッシュから静かに削除。オフライン・アップロード失敗・競合時は実行しない
- **名前変更/削除への追従**: タイトル名変更でJSONと一緒に`/{タイトル名}/`フォルダをmoveし、キャッシュキーも付け替え。作品削除で画像フォルダとキャッシュも削除

## 同期仕様(Phase 3・将来の自分向けメモ)

- **構成**: [storage-bridge](https://github.com/MoyoriEki/storage-bridge)を`<script>`で読み込み(無改造)。rev取得(`files/get_metadata`)だけはブリッジ未提供のため、`plotboard.html`側に`_apiFetch`を使う薄いラッパー(`dbxMeta`)がある。保存先は Dropbox `/PlotBoard/{タイトル名}.json`
- **アップロードのトリガ**: ①編集後20秒デバウンス ②ボード画面の⟳ボタン ③一覧に戻るとき ④タブが裏に回るとき(visibilitychange) ⑤オンライン復帰(onlineイベント)・Dropbox接続時は未同期(dirty)の全作品を順次アップロード
- **競合検知**: 最後に同期したrevを作品ごとに保持し、アップロード直前に現在のrevと比較。不一致=他端末の更新 → ダイアログで「この端末の内容で上書き」「クラウドの内容を読み込む(この端末の変更は破棄)」「あとで決める(保留)」の選択。自動マージはしない(Last-Write-Wins)
- **開くとき**: クラウドのrevがローカルと違い、ローカルに未同期変更が無ければ黙ってダウンロードして置き換え。未同期変更があれば競合ダイアログ
- **オフライン時**: ローカル編集は通常どおり。名前変更・削除は同期済み作品に対しては不可(Dropbox側の反映を伴うため。「次回同期時に反映」方式はキュー管理が増えるので採用しない)。ローカルのみの作品は制限なし
- **マイグレーション**: Phase 2までの単一ボードデータは初回起動時に「無題」として取り込み。旧データ(IndexedDBの`docs`ストア)は保険として残したまま
- **インジケータ**: ローカル保存済み / クラウド同期済み / 同期待ち / 同期中 / オフライン / 競合 / 保存中
