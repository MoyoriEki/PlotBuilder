# PlotBuilder

漫画・物語のプロットをグリッドスナップ式ブロックとして配置・組み換えするWebアプリ「プロットボード」。

- 本体: `plotboard.html`(単一HTMLファイル・ノービルド)
- 公開URL: https://moyorieki.github.io/PlotBuilder/plotboard.html (GitHub Pages / mainへのpushで自動デプロイ)

## 設計方針

- 軽量・直感優先。D&DはPointer Eventsによる自前実装
- **縦長ファースト**: モバイル(スマホ/タブレット縦持ち)を主要ユースケースとし、初期グリッドは縦長(6列×12行)。今後のUIも縦長画面を優先して設計する
- ローカル(IndexedDB)が常に一次ストレージ。Dropbox連携は任意(未接続でも全機能が使える)

## 同期仕様(Phase 3・将来の自分向けメモ)

- **構成**: [storage-bridge](https://github.com/MoyoriEki/storage-bridge)を`<script>`で読み込み(無改造)。rev取得(`files/get_metadata`)だけはブリッジ未提供のため、`plotboard.html`側に`_apiFetch`を使う薄いラッパー(`dbxMeta`)がある。保存先は Dropbox `/PlotBoard/{タイトル名}.json`
- **アップロードのトリガ**: ①編集後20秒デバウンス ②ボード画面の⟳ボタン ③一覧に戻るとき ④タブが裏に回るとき(visibilitychange) ⑤オンライン復帰(onlineイベント)・Dropbox接続時は未同期(dirty)の全作品を順次アップロード
- **競合検知**: 最後に同期したrevを作品ごとに保持し、アップロード直前に現在のrevと比較。不一致=他端末の更新 → ダイアログで「この端末の内容で上書き」「クラウドの内容を読み込む(この端末の変更は破棄)」「あとで決める(保留)」の選択。自動マージはしない(Last-Write-Wins)
- **開くとき**: クラウドのrevがローカルと違い、ローカルに未同期変更が無ければ黙ってダウンロードして置き換え。未同期変更があれば競合ダイアログ
- **オフライン時**: ローカル編集は通常どおり。名前変更・削除は同期済み作品に対しては不可(Dropbox側の反映を伴うため。「次回同期時に反映」方式はキュー管理が増えるので採用しない)。ローカルのみの作品は制限なし
- **マイグレーション**: Phase 2までの単一ボードデータは初回起動時に「無題」として取り込み。旧データ(IndexedDBの`docs`ストア)は保険として残したまま
- **インジケータ**: ローカル保存済み / クラウド同期済み / 同期待ち / 同期中 / オフライン / 競合 / 保存中
